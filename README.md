## 🧭 Philosophy / 思维模型

> I don’t optimize for code. I optimize for systems that compound.

### 1. System Thinking First
- Any feature is a system design problem in disguise
- Latency, cost, reliability are first-class constraints

### 2. Infra → Product Mapping
- Infrastructure is only meaningful when it creates product leverage
- Every system must answer: *what capability does it unlock?*

### 3. Constraint-Driven Engineering
- GPU/CPU cost is not a detail, it's the product boundary
- Scalability decisions are made at design time, not after failure

### 4. Abstraction Discipline
- Every abstraction must reduce long-term complexity, not hide it
